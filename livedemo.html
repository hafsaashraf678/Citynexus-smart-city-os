System Architecture

🔄 Process State Diagram

Program Journey: When any service (process) starts, it goes through 5 states. Each color shows a specific state.

NEW - Process Created
READY - Waiting for CPU
RUNNING - Executing
WAITING - I/O or Event
TERMINATED - Completed

① NEW State

What happens: Process is just created, memory is being allocated.

Example: When you press Emergency Service button.

② READY State

What happens: Process is ready, waiting for CPU time. It's in Ready Queue.

Example: Traffic System waiting for CPU.

③ RUNNING State

What happens: Process is executing on CPU. Active work is happening.

Example: Security System scanning cameras.

④ WAITING State

What happens: Process is waiting for I/O operation (disk, network, etc).

Example: Utility system loading data from database.

⑤ TERMINATED State

What happens: Process completed its work, memory is freed.

Example: Transport service finished its task.

🔀 Important Transitions (State Changes)

1. admit: NEW → READY
Process is loaded into memory
2. dispatch: READY → RUNNING
Scheduler allocates CPU
3. I/O request: RUNNING → WAITING
Input/Output operation needed
4. I/O complete: WAITING → READY
I/O done, back to queue
5. preempt/timeout: RUNNING → READY
Time slice ended or higher priority
6. exit: RUNNING → TERMINATED
Process completed

Key Features & Innovations

Real-Time Processing
